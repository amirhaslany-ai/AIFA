## Purpose

Meeting notes, using a consistent template so they're searchable and so action items don't get lost. Currently structure-only; use `TEMPLATE_Meeting_Notes.md`.

## Update rules

- One file per meeting, named `YYYY-MM-DD_Short-Topic.md`, not one running log — makes individual meetings linkable and archivable independently.
- Action items that turn into real decisions get promoted to the appropriate `12_Decisions/` log, not left only in the meeting notes — meeting notes are the record of *discussion*, decision logs are the record of *outcome*.
- File a meeting's notes here even if nothing was decided — a record of "we discussed X and didn't resolve it" is still useful (it's effectively a dated `Open questions` entry).

## Ownership

Unassigned.

## Relationships

Action items flow into `12_Decisions/` (when a decision results) or into the relevant strategy folder (`01_Product/`, `02_Business/`, etc.) as follow-up work.
